## RBAC 问题修复与测试账号

### 发现的问题与修复

| 问题 | 修复方案 |
|------|----------|
| `migrate_rbac.sql` 中 `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` 在 MySQL 9.6 中不兼容 | 改为存储过程 `add_column_if_not_exists()`，先查 `information_schema.COLUMNS` 再添加列 |
| `pom.xml` 缺少 `spring-boot-starter-aop` 依赖 | 已添加 `spring-boot-starter-aop` dependency |
| `JwtUtil.java` 中 `getExpirationDate()` 方法声明丢失，只剩孤立方法体 | 重新补全方法声明；并补充缺失的 `java.util.List/Arrays/Collections` 导入 |
| `AuthController.refresh()` 从 refresh token 中读取 `role`（refresh token 不存 role） | 改为从数据库读取 `user.getRole()` 再规范化，避免刷新后角色变成 EMPLOYEE |
| `Permission.java` 实体缺少 `children` 字段，权限树无法显示子权限 | 添加 `children: List<Permission>` 字段（`@TableField(exist=false)`） |
| `PermissionServiceImpl.getTree()` 未把子权限挂到父节点 | 修复为 `root.setChildren(children)`，并兼容 `parentId` 为 null 的情况 |
| `RoleManageView.vue` 使用 `permissionApi.tree()` 但未导入 | 从 `@/api/role` 导入 `permissionApi` |
| `App.vue` 侧边栏缺少「角色管理」入口 | 在系统管理分组下添加 `/role-manage` 菜单项，权限码 `system:role` |
| 测试账号需要创建 | 在 `migrate_rbac.sql` 中加入 5 个测试账号，并已执行到数据库 |
| 遗留 `PermissionAspect.java.bak` 备份文件 | 已删除，避免被 Maven 编译 |
